范式：<此次提交的类型><细化范围(可选)>: <对提交的描述>

|       类型 | 描述            |
| -------: | :------------ |
|     feat | 新功能           |
|      fix | 修复bug         |
|     docs | 文档变更          |
|    style | 代码格式改变        |
| refactor | 某个已有功能重构      |
|     perf | 性能优化          |
|     test | 增加测试          |
|   revert | 撤销上一次的 commit |
|    chore | 找不到哪一类就用这一类   |

比如你对某前端项目的图片获取功能进行了重构：

```bash
git commit -m 'refactor(图片获取): 重构图片加载逻辑'
```
