# “视界之声”智慧盲人助手开发文档

## 1. 文档目的

本文档面向“视界之声”智慧盲人助手项目的评审、验收与交付场景，重点说明新应用结构与开发环境、主要功能 UI 原型、软硬件配置、高层设计、共享数据样例，以及数据流转与展示方式。由于快应用形态已完整覆盖安卓端核心交互流程，本文档以快应用方案为主要说明对象。

## 2. 新应用结构及开发环境设计

### 2.1 新应用结构

项目采用“快应用外壳 + 后端 Web 服务 + AI 能力模块”的分层结构，核心组成如下：

1. 前端快应用工程位于 [front-end/VisionVoiceProject](front-end/VisionVoiceProject)，负责登录、跳转、账号态缓存和 WebView 容器控制。
2. 业务主界面由后端 Flask 提供，页面模板位于 [back-end/templates](back-end/templates)，静态资源位于 [back-end/static](back-end/static)。
3. 后端主入口为 [back-end/run.py](back-end/run.py)，负责认证、聊天流式输出、图片管理、有声相册、验证码发送与语音处理。
4. 图像识别、语音识别、语音合成和短信能力由独立模块承担，分别位于 [back-end/agent_files](back-end/agent_files) 与 [back-end/service](back-end/service)。
5. 用户数据、验证码、图片和音频等持久化数据分别保存在 [back-end/static/user.json](back-end/static/user.json)、[back-end/configs/verification_code_dict.json](back-end/configs/verification_code_dict.json) 和 [back-end/user_images](back-end/user_images) 下。

运行链路上，快应用完成登录后通过 WebView 载入后端页面；后端页面再通过 JWT、Cookie、Socket.IO 和 HTTP 接口与服务端交互，形成统一的业务体验。

### 2.2 应用模块划分

#### 前端快应用层

- 登录注册页：负责短信验证码登录、账号密码登录、账号注册。
- 内容承载页：负责 WebView 载入后端页面，并将 token、username、nickname 等信息同步到 Cookie。
- 通用工具层：封装网络请求、本地存储、Cookie 操作和页面跳转能力。

#### 后端业务层

- 身份认证与会话层：处理注册、登录、token 校验、退出登录。
- 智能体对话层：支持文字、语音、视频多模态交互，以及流式输出。
- 生活助手层：支持避障、寻物、环境识别、定位等功能。
- 有声相册层：支持图片导入、图片解析、图片描述音频生成、语速调整与图片问答。
- 资源管理层：负责图片、音频、文本和配置文件的存取与维护。

### 2.3 开发环境设计

本项目采用“本地开发机 + 移动终端测试 + 后端服务联调”的环境设计，重点保证开发效率、无障碍适配和多模态交互验证。

#### 环境分工

- 开发机：用于编写前端、后端与算法代码，完成页面调试、接口联调和资源管理。
- 移动终端：用于验证登录注册、语音交互、拍照识别、定位、寻物和有声相册等真实使用场景。
- 服务端运行环境：用于承载 Flask 业务接口、Socket.IO 流式通信、文件读写和第三方能力调用。

#### 设计原则

- 优先保证无障碍交互链路完整，核心页面应适配读屏与语音输入。
- 前后端联调应保持同一套会话标识与资源路径，避免多端状态不一致。
- 图片、音频和聊天记录要按用户隔离保存，便于测试与交付。
- 开发环境与演示环境尽量保持一致，降低部署后路径或权限差异带来的风险。

#### 配置文件与运行约定

- 第三方接口配置集中在 [back-end/static/api.json](back-end/static/api.json) 中，实际交付时应使用安全的环境配置或密钥注入方式管理，不应在文档或代码仓库中明文暴露生产密钥。
- 语音参数与音频设置集中在 [back-end/configs/audio_settings.json](back-end/configs/audio_settings.json)。
- 验证码字典写入 [back-end/configs/verification_code_dict.json](back-end/configs/verification_code_dict.json)。

## 4. 作品使用的软件和硬件

### 4.1 软件清单

#### 后端软件

- Python 3.11.4：后端运行环境。
- Flask 3.0.3：提供 Web 服务与接口路由。
- Flask-SocketIO 5.4.1：支持流式输出与异步事件推送。
- Flask-CORS 5.0.0：处理跨域访问控制。
- flask-jwt-extended 4.6.0：提供 token 认证能力。
- zhipuai 2.1.5.20230904：用于大模型调用。
- OpenCV、Ultralytics、TensorFlow、NumPy、scikit-learn、samplerate、cvzone：用于视觉识别、图像处理和音频处理。
- alibabacloud_dysmsapi20170525：用于短信验证码服务。

#### 前端软件

- 快应用开发工具：用于页面开发、打包与调试。
- Node.js 与 Yarn：用于前端依赖管理和构建。
- 快应用运行容器或支持 WebView 的手机系统环境：用于端侧运行与联调。

#### 运行与调试工具

- 浏览器：用于访问后端 Web 页面和接口联调。
- 日志查看工具：用于排查 Flask 与 Socket.IO 输出问题。
- 图片查看器：用于核对有声相册和寻物画廊资源。

### 4.2 硬件清单

- 智能手机或平板：至少支持摄像头、麦克风、扬声器、定位与网络连接，是系统的主要运行终端。
- 摄像头：用于避障、寻物、环境识别和有声相册图片导入。
- 麦克风：用于语音输入和录音识别。
- 扬声器或耳机：用于语音播报与聊天输出。
- 开发电脑：用于运行后端服务、前端构建、模型调试和资源管理。

### 5.1 总体系统架构

系统采用前后端分离并结合多模态 AI 能力的架构。前端负责无障碍交互和页面承载，后端负责业务逻辑、鉴权、文件存储、模型调用与流式通信，第三方服务负责短信、地图和大模型能力。系统主要包含语音视频对话、辅助避障和帮我寻物三类核心交互，相关架构如下：

### 5.2 主要模块设计

#### 5.2.1 身份认证模块

负责注册、登录、短信验证码发送、token 校验和退出登录。系统支持手机号验证码登录和账号密码登录两种模式。注册成功后生成 access token，并将用户名、昵称、手机号写入用户信息文件。

#### 5.2.2 智能体对话模块

负责生活助手和心灵树洞两个智能体的多模态对话能力。后端根据当前智能体名称读取用户历史会话，调用大模型生成流式回复，并将结果写回聊天记录。文字、语音和视频输入统一映射为对话消息，以降低前端处理复杂度。

#### 5.2.3 生活助手模块

负责避障、寻物、环境识别和帮我定位等功能。避障依赖摄像头与单目视觉分析；寻物依赖目标图库与实时识别；环境识别依赖图像理解；定位依赖地理位置数据与地图接口能力。

#### 5.2.4 有声相册模块

负责图片上传、图片描述生成、语音合成、语速保存、图片文本查询与图片删除。该模块保持图片、文本、音频三类数据同步维护，便于视障用户通过听觉完成相册浏览。

#### 5.2.5 资源与缓存模块

负责缓存图片、音频、对话与临时中间文件。前端通过本地存储记录 token、用户名和昵称；后端通过 .cache 目录维护临时图片与音频文件，避免多轮多模态任务之间互相覆盖。

## 6. 共享数据样例

### 6.1 用户主数据

用户信息统一存放在 [back-end/static/user.json](back-end/static/user.json)，核心字段包括：

- username：登录用户名。
- password：登录密码。
- nickname：显示昵称。
- phone：手机号。
- agents：各智能体的聊天历史。

示例结构如下：

```json
{
	"username": "demo_user",
	"password": "******",
	"nickname": "小天",
	"phone": "13800000000",
	"agents": {
		"defaultAgent": {
			"chat_history": [
				{"role": "system", "content": "..."},
				{"role": "assistant", "content": "..."}
			]
		},
		"psychologicalAgent": {
			"chat_history": [
				{"role": "assistant", "content": "..."}
			]
		}
	}
}
```

### 6.2 验证码数据

验证码写入 [back-end/configs/verification_code_dict.json](back-end/configs/verification_code_dict.json)，按手机号保存验证码和时间戳，用于注册和登录校验。

### 6.3 图片与音频数据

#### 寻物图片

- 存储目录：back-end/user_images/<用户名>/item_images/
- 数据类型：用户手动拍摄并命名的目标图片。
- 典型用途：寻物模式中的目标物品选择与识别参考。

#### 有声相册图片

- 存储目录：back-end/user_images/<用户名>/album/images/
- 数据类型：相册图片原图。
- 关联文本：back-end/user_images/<用户名>/album/texts/
- 关联音频：back-end/user_images/<用户名>/album/audios/
- 语速配置：back-end/user_images/<用户名>/album/talk_speed_config.json

#### 临时缓存数据

- 存储目录：back-end/.cache/<用户名>/
- 数据类型：语音识别临时 wav、实时拍照图片和多人图像对话缓存。

### 6.4 配置数据

- 第三方接口配置：back-end/static/api.json
- 音频参数配置：back-end/configs/audio_settings.json
- 用户公告与协议：back-end/configs/user_announcement.txt、back-end/configs/user_agreement.txt

## 7. 数据流转及展示方式

### 7.1 登录与会话流转

1. 用户在快应用登录页输入手机号验证码或用户名密码。
2. 前端通过接口请求后端登录服务。
3. 后端校验用户信息后生成 JWT token，并返回用户信息。
4. 快应用将 token、username、nickname 写入本地存储。
5. 快应用进入内容页后，将 token 同步到 WebView Cookie。
6. 后端页面通过 token 完成鉴权，进入智能体或功能页面。

展示方式：登录页展示输入框、验证码获取按钮和登录注册切换；登录成功后默认进入生活助手聊天界面。

### 7.2 智能体对话流转

1. 用户输入文字，或通过语音按钮录入语音。
2. 前端将语音或文字转换为请求数据，上传后端。
3. 后端根据当前智能体和历史聊天记录构造 prompt。
4. 大模型以流式方式输出结果。
5. 后端边生成边发送到前端，前端同步显示文字并触发语音播报。
6. 完整回复结束后，后端将结果写回用户聊天历史。

展示方式：对话区实时滚动显示文本，语音播放区按句播报，系统支持随时停止输出。

### 7.3 避障与环境识别流转

1. 用户开启摄像头权限并进入生活助手功能。
2. 前端采集实时图像或拍照结果。
3. 避障功能在端侧完成视觉分析，环境识别场景将拍照结果发送至后端进行理解。
4. 系统输出障碍物提示、环境描述或位置建议。
5. 前端以语音提示为主，辅以文字状态展示。

展示方式：以大字号按钮、图标和语音提示为主，避免依赖复杂视觉元素。

### 7.4 寻物流转

1. 用户先在“寻物画廊”中保存目标物品图片并命名。
2. 用户发起寻物请求后，前端展示可选目标列表。
3. 用户选择目标图片后，端侧以该图片为参考完成目标匹配与识别。
4. 系统输出识别结果与目标位置提示。
5. 前端通过语音和文本同步提示用户。

展示方式：列表以缩略图、名称和操作按钮呈现，支持新增、重命名、删除和查看等操作。

### 7.5 有声相册流转

1. 用户从摄像头或相册导入图片。
2. 后端保存图片原图，并调用图像理解能力生成描述文本。
3. 后端调用语音合成接口生成对应音频。
4. 图片、文本、音频和语速配置分别落盘。
5. 用户点击图片时，前端请求文本或音频接口进行展示与播报。

展示方式：相册页面以图片网格或列表展示，图片旁标注“已生成解析”状态；点击后打开详情页，可查看文字解析并播放音频。

### 7.6 数据管理与安全流转

1. 用户信息、聊天记录和图片资源按用户名隔离存储。
2. 认证信息使用 token 和 Cookie 双通道传递，降低跨页面丢失风险。
3. 敏感配置与验证码数据集中保存，并通过服务端权限控制读取。
4. 图片与音频临时文件在任务结束后及时清理，减少缓存污染。

展示方式：数据管理过程对用户透明，前端仅展示必要状态信息；敏感数据不直接暴露在界面层。
